Introduction

This is API Docs for a API E-Commerce Application. Here, you will find All the Endpoints in this API.

This is the URL where the API is hosted

The URL is:

  • https://apus-ecommerce-api.onrender.com
Example Usage
Below is the Simple Example how you can use this library to fetch all Products available in the database used in backend for this API: // Please Note: This Example Code uses axois Library and the Below code is in JavaScript const res = await axios.get(https://apus-ecommerce-api.onrender.com/api/product/products); Outcome: res.data.products will give you all the products available in database
Auth Routes
  • /api/signup: Register the User

    A fetch Request for Register will look like: // Note: All the options in Body is Required i.e. firstname, lastname, email, username and password (8 characters or more) const options = { method: 'POST', body: '{ "firstname":"John", "lastname":"Smith", "email":"test@gmail.com", "username":"user name", "password":"************"}' }; fetch('https://apus-ecommerce-api.onrender.com/api/signup', options) .then(response => response.json()) .then(response => console.log(response)) .catch(err => console.error(err)); Returns a JWT Token, which needs to be decoded.

  • /api/login: Login the User

    A fetch Request for login will look like: // Note: All the options in Body is Required i.e. email and password (8 characters or more) const options = {method: 'POST', body: '{"email":"test@gmail.com", "password":"**********"}'}; fetch('https://apus-ecommerce-api.onrender.com/api/login', options) .then(response => response.json()) .then(response => console.log(response)) .catch(err => console.error(err)); Returns a JWT Token, which needs to be decoded.

  • /api/users/:userId: Update the User

    A fetch Request for Updating the User will look like: // Note: All the options in Body is Required i.e. email and password (8 characters or more) const options = { method: 'POST', headers: {Authorization: 'Bearer JWT_TOKEN_HERE', 'Content-Type': 'application/json'}, body: '{ "firstname":"test@gmail.com", "lastname":"test@gmail.com", "username":"test@gmail.com"}' }; fetch('https://apus-ecommerce-api.onrender.com/ api/users/63a8cf12981d84d33d8bfcc2', options) .then(response => response.json()) .then(response => console.log(response)) .catch(err => console.error(err));

  • /api/verify-email: Verify the User Path Email. This path is automatically used during Registration

    Example will look like // Note: The verificationString will come from the email received when registering the user const options = { method: 'PUT', body: '{"verificationString":"8b45a65b-c624-4a93-8cf5-344e4321203d"}' }; fetch('https://apus-ecommerce-api.onrender.com/api/verify-email', options) .then(response => response.json()) .then(response => console.log(response)) .catch(err => console.error(err));

  • /api/forgot-password/:email Forget Password Email. Recieve Product Password Email to Update password if forgetten
  • /api/users/:passwordResetCode/reset-password Reset the password
  • /api/users/delete/:id Delete the User
  • /api/users Gives all the Users
  • /api/users/stats Give stats of Users
Product Routes
  • /api/product/products : Get All Products. This End Support further query on the basic of URL query

    Request for Getting all the Products: const options = {method: 'GET'}; fetch('https://apus-ecommerce-api.onrender.com', options) .then(response => response.json()) .then(response => console.log(response)) .catch(err => console.error(err));

  • /api/product/create Create a New Product only if you are logged in as Admin
  • /api/product/delete/:id Delete the Product
  • /api/product/update/:id Update the Product
Order Routes
  • /api/orders/create : Create a new Order
  • /api/orders Get All the Orders
  • /api/orders/income Gives the monthly income from all the Orders
Payment Routes
  • /api/stripe/payment Makes a payment
Contact Routes
  • /api/contact: Send the Owner that is me (Amrit) the query of Customer
